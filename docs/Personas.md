# Personas — 사용자 페르소나

> IA.md / UserScenarios.md와 독립 관리.

## P-01 준호 — 개발자이자 시스템 소유자 (주 사용자)

- **프로필**: 30대 소프트웨어 개발자. 이 시스템을 직접 만들고 운영한다.
  Claude API와 Python에 익숙하다. Windows 로컬 환경에서 개발·사용.
- **목표**:
  - 혼자 하기엔 관점이 부족한 작업(아이디어 평가, 문서 초안, 비교 분석)을
    여러 역할의 에이전트에게 맡기고 품질 높은 결과를 얻는다.
  - 에이전트들이 실제로 어떻게 협업하는지 과정을 관찰하고 팀 구성을 개선한다.
- **불편/우려**:
  - API 비용이 통제 없이 새는 것 (→ 토큰 예산, 사용량 가시화 필수)
  - 에이전트들이 결론 없이 맴도는 것 (→ 종료 정책, 취소 기능)
- **행동 특성**: YAML 설정 직접 편집을 꺼리지 않음. 문제가 생기면 로그를 직접 본다.

## P-02 수민 — 비개발자 동료 (부 사용자, 확장 시나리오)

- **프로필**: 기획/마케팅 직군. 준호가 서버를 띄워주면 브라우저로만 사용한다.
  API, YAML, 터미널을 다루지 않는다.
- **목표**: 태스크를 문장으로 입력하고 결과만 받으면 된다. 과정은 "진행 중이구나"
  정도만 확인.
- **불편/우려**:
  - 에러 메시지가 기술 용어로 나오면 대응 불가 (→ 사용자 노출 메시지는 평이하게)
  - 결과가 어디 있는지 못 찾는 것 (→ 완료 시 결과 카드 강조, 세션 목록에서 재접근)
- **시사점**: 초기 버전은 P-01 중심으로 설계하되, 대시보드의 제출→결과 경로는
  P-02도 헤매지 않을 수준으로 단순해야 한다.
